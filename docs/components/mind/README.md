# Mind（精神）コンポーネント

Mind コンポーネントは、AI Tuber の根幹となる「キャラクター定義」を担う層です。
本システムはシステムロジック（Saint Graph）とキャラクター定義（Mind）を完全に分離しているため、Mind を構成するファイル群を差し替えるだけで、自由に新しいキャラクターを追加・変更することができます。

---

## 1. キャラクターを構成する要素

キャラクターを作成・変更するには、`data/mind/[キャラクター名]/` 配下に以下のファイルを用意します。

### `persona.md`
キャラクターのロールプレイを規定する中核プロンプトです。性格、口調、コアとなるアイデンティティや、ニュース配信などの際の振る舞い（Few-Shot Example 含む）を記述します。
Gemini Flash Lite などの最新モデルはプロンプトの追従性が非常に高いため、以前のように JSON 出力をプロンプト側で強く強制したりする必要はなく、キャラクターのセリフの記述に注力できます。

### `mind.json`
システムがキャラクターを認識・制御するためのメタデータです。
例えば、VOICEVOX で使用する話者 ID (`speaker_id`) などをここで定義します。

### アセット画像・動画 (`assets/`)
キャラクターの「外見と表現」を規定するファイル群です。
これらは OBS を動かしている Body (肉体) コンテナによって配信時に読み込まれます。

---

## 2. アセット（表情差分・モーション）の作成手法

本リポジトリで推奨・実践している、2026年現在のAI技術を活用したモダンな立ち絵・表情差分の作成プロセスを紹介します。

### Qwen-Image-Edit-Rapid-AIO によるベースイラストと表情生成
過去のように Stable Diffusion で `Inpaint` 等を駆使して複雑に表情差分を作る時代は終わり、現在は自然言語（日本語）ベースの指示で完結する **[Qwen-Image-Edit-Rapid-AIO](https://huggingface.co/Phr00t/Qwen-Image-Edit-Rapid-AIO)** を活用しています。

- **特徴**: プロンプトで「服装を変えて」「〜な表情にして」と指示するだけで、ベースとなるイラストから一貫性を保ったまま各種差分画像が生成できます。
- **UI**: 構築には **ComfyUI** を利用します。ノードベースで難しく見えがちですが、コミュニティで配布されているテンプレート（ワークフロー）をそのまま読み込むことで、複雑な配線を意識することなく利用できます。

### Sora によるモーション作成（リップシンクと瞬き）
AI Tuber の表現をより豊かに（Live2D的に）するため、生成動画を活用します。

- 単純な静止画差分ではなく、**[Sora](https://openai.com/sora)** 等の動画生成AIでキャラクターを 2〜3 秒動かした短い動画（MP4 / WebM）を生成します。
- それらを無限ループ用の動画素材として `assets/` に配置します。
- 配信中、Saint Graph が `[emotion: joyful]` のように感情タグを出力すると、OBS は瞬時にその感情に対応したショート動画に切り替えてループ再生します。これにより、Live2D モデルを別途作成せずとも、リップシンクや瞬きなどの自然なアニメーションを伴う配信が実現できます。

> **OBS 側の設定注意点**: 動画を素材として利用する場合、OBS のメディアソース設定で「ループ」をオンにし、「非アクティブ時にファイルを閉じる」をオフにすることを推奨します（映像の切り替え時の一瞬の暗転を防ぐため）。

## 3. 実稼働における Mind のロードプロセス

設計原則である**ステートレスコンテナ**の概念に基づき、Mind のデータ群は本体の Docker イメージには埋め込まれません。

1. **配置**: `cloudbuild-mind.yaml` 等により、キャラクターのデータは Google Cloud Storage (GCS) 等にアップロードされます。
2. **起動時**:
   - `Saint Graph` 起動時: `StorageClient` が指定されたキャラクター（環境変数 `CHARACTER_NAME`）の `persona.md` や `mind.json` を GCS から読み込み、コンテキストとして LLM にロードします。
   - `Body` (OBS コンテナ) 起動時: `download_assets.py` が GCS から `assets/` 配下の画像・動画一式をローカル (`/app/assets/`) にダウンロードし、OBS がそれを参照して立ち絵を表示します。

これにより、コンテナを一切再ビルドすることなく、起動時のパラメータ変更だけで別の AI Tuber を配信させることが可能になっています。
